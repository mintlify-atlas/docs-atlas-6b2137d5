## Installation

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/truthixify/ekg
    cd ekg
    ```
  </Step>

  <Step title="Configure environment variables">
    Copy the example environment file and add your Gemini API key:
    
    ```bash
    cp .env.example .env
    ```
    
    Edit `.env` and set your API key:
    
    ```bash .env
    GEMINI_API_KEY=your_gemini_api_key_here
    NEO4J_URI=bolt://localhost:7687
    NEO4J_USER=neo4j
    NEO4J_PASSWORD=password
    ```
    
    <Warning>
      Keep your `.env` file secure and never commit it to version control. The `.gitignore` file already excludes it.
    </Warning>
  </Step>

  <Step title="Start the system">
    Launch Neo4j and the EKG application:
    
    ```bash
    docker-compose up
    ```
    
    This will:
    - Start Neo4j graph database on ports 7474 (web UI) and 7687 (bolt)
    - Parse configuration files from the `data/` directory
    - Build the knowledge graph
    - Start the web interface at http://localhost:8000
    
    <Note>
      First startup takes 1-2 minutes while Docker builds the images and Neo4j initializes.
    </Note>
  </Step>

  <Step title="Verify the installation">
    Check that all services are healthy:
    
    ```bash
    curl http://localhost:8000/api/health
    ```
    
    Expected response:
    ```json
    {
      "status": "healthy",
      "components": {
        "storage": true,
        "query_engine": true,
        "query_parser": true,
        "neo4j": true
      }
    }
    ```
  </Step>
</Steps>

## Understanding the data

EKG parsed the sample configuration files in the `data/` directory:

### Docker Compose services

The system loaded 7 services from `data/docker-compose.yml`:

<CodeGroup>
```yaml api-gateway
api-gateway:
  build: ./services/api-gateway
  ports:
    - "8080:8080"
  environment:
    - AUTH_SERVICE_URL=http://auth-service:8081
    - ORDER_SERVICE_URL=http://order-service:8082
    - PAYMENT_SERVICE_URL=http://payment-service:8083
  depends_on:
    - auth-service
    - order-service
    - payment-service
  labels:
    team: platform-team
    oncall: "@alice"
```

```yaml payment-service
payment-service:
  build: ./services/payment-service
  ports:
    - "8083:8083"
  environment:
    - DATABASE_URL=postgresql://postgres:secret@payments-db:5432/payments
    - NOTIFICATION_SERVICE_URL=http://notification-service:8085
  depends_on:
    - payments-db
  labels:
    team: payments-team
    oncall: "@frank"
    pci_compliant: "true"
```
</CodeGroup>

EKG extracts:
- Service names and types
- Dependencies from `depends_on` and environment variables
- Team ownership and oncall information from labels
- Port mappings and configuration

### Team ownership

From `data/teams.yaml`, EKG knows which teams own which services:

```yaml data/teams.yaml
teams:
  - name: platform-team
    contact: "@alice"
    owns:
      - api-gateway
      - notification-service
      - redis-main
  
  - name: payments-team
    contact: "@frank"
    owns:
      - payment-service
      - payments-db
```

## Exploring the graph

You can visualize the knowledge graph in Neo4j Browser:

<Steps>
  <Step title="Open Neo4j Browser">
    Navigate to http://localhost:7474
    
    Login with:
    - Username: `neo4j`
    - Password: `password`
  </Step>

  <Step title="Run Cypher queries">
    Try these queries to explore the graph:
    
    ```cypher
    // Show all services
    MATCH (n {type: 'service'})
    RETURN n
    LIMIT 25
    ```
    
    ```cypher
    // Show service dependencies
    MATCH (s {type: 'service'})-[r]->(d)
    RETURN s, r, d
    LIMIT 50
    ```
    
    ```cypher
    // Show team ownership
    MATCH (t {type: 'team'})-[:OWNS]->(asset)
    RETURN t, asset
    ```
  </Step>
</Steps>

## Adding your own data

To analyze your own infrastructure:

<Steps>
  <Step title="Replace the sample data">
    Put your configuration files in the `data/` directory:
    
    ```bash
    data/
    ├── docker-compose.yml    # Your Docker Compose services
    ├── teams.yaml            # Your team ownership info
    └── k8s-deployments.yaml  # (Optional) Kubernetes resources
    ```
  </Step>

  <Step title="Reload the data">
    Trigger a reload via the API:
    
    ```bash
    curl -X POST http://localhost:8000/api/reload
    ```
    
    Or restart the Docker containers:
    
    ```bash
    docker-compose restart ekg-app
    ```
  </Step>

  <Step title="Verify the new data">
    Check that your services were loaded:
    
    ```bash
    curl http://localhost:8000/api/entities
    ```
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Container won't start">
    Check Docker logs:
    ```bash
    docker-compose logs ekg-app
    docker-compose logs neo4j
    ```
    
    Common issues:
    - Missing `GEMINI_API_KEY` in `.env` file
    - Port 8000 or 7687 already in use
    - Insufficient Docker memory (Neo4j needs ~512MB)
  </Accordion>

  <Accordion title="Health check fails">
    Wait for Neo4j to fully initialize (30-60 seconds on first start), then check:
    
    ```bash
    docker-compose ps
    ```
    
    All services should show "healthy" status.
  </Accordion>

  <Accordion title="Queries return no results">
    Verify data was loaded:
    
    ```bash
    docker-compose logs ekg-app | grep "Total loaded"
    ```
    
    You should see: `Total loaded: X nodes and Y edges`
    
    If not, check that your configuration files exist in `data/`.
  </Accordion>

  <Accordion title="LLM errors">
    Verify your Gemini API key is valid:
    
    ```bash
    curl https://generativelanguage.googleapis.com/v1/models?key=YOUR_API_KEY
    ```
    
    Check rate limits at [Google AI Studio](https://aistudio.google.com/).
  </Accordion>
</AccordionGroup>
